**Hedef**: Listing schema'ya fabrika ürün kontrolleri.

| Dosya | İşlem |

| `packages/shared/shared-persistence/src/schemas/backend/listing.schema.ts` | 5 yeni alan: `visibleTo` (enum `ALL_DEALERS\|SELECTED_DEALERS\|NONE`), `availableFrom?`, `availableTo?`, `allowOnlineResale` (default `false`), `maxOrderQtyPerDealer` (ekosistem listing'de zorunlu), `selectedDealerIds?` |
| `apps/backend/src/modules/listing/dto/create-listing.dto.ts` | DTO + class-validator |
| `apps/backend/src/modules/listing/application/commands/create-listing.handler.ts` | Validation: ekosistem listing ise `maxOrderQtyPerDealer` null olamaz |
| `apps/backend/src/modules/listing/application/queries/list-listings.handler.ts` | `visibleTo` + `availableFrom/To` ile filter **+ anonymousId mapping (blind pool kuralı §5.3 / barter-rules §5)** — `vendorId` ve `dealerId` serializer'da `anonymousId` ile maskelenir |
| `apps/backend/src/modules/listing/dto/listing-response.dto.ts` | **YENİ**: `class-transformer` ile `@Expose({ name: 'anonymousId' })` — orijinal ID'ler `@Exclude()` |
| `apps/backend/src/modules/barterborsa/application/services/anonymizer.service.ts` | **YENİ**: `vendorId → anonymousId` deterministik hash (HMAC-SHA256 + tenant secret) |

**Acceptance**:
- Ekosistem listing oluştururken `maxOrderQtyPerDealer` zorunlu (null → 400 hata)
- `visibleTo=SELECTED_DEALERS` ise `selectedDealerIds.length > 0` zorunlu
- `availableFrom < availableTo` (her ikisi varsa)
- **Ekosistem listing response'unda `vendorId`/`dealerId` görünmez; yerine `anonymousId` (deterministik hash) döner**
- Sipariş onaylandıktan sonra (escrow tamamlanınca) gerçek kimlik açılır

## 🔴 P0-2: Garaj Günü (Flash Sale) Modülü (§4.4) — YENİ MODÜL

**Hedef**: Atomic stok decrement + otomatik kapanma.

| Dosya | Tip |
|---|---|
| `packages/shared/shared-persistence/src/schemas/backend/garageSale.schema.ts` | Yeni schema (factoryId, productId, discountRate, maxTotalQty, soldQty, maxQtyPerDealer, startsAt, endsAt, status: `SCHEDULED\|ACTIVE\|EXHAUSTED\|EXPIRED`) |
| `apps/backend/src/modules/garage-sale/garage-sale.module.ts` | Module |
| `…/domain/garage-sale.entity.ts` | Domain entity |
| `…/application/commands/create-garage-sale.{command,handler}.ts` | Fabrika oluşturma (APEX kontrol) |
| `…/application/commands/purchase-from-garage-sale.{command,handler}.ts` | `findOneAndUpdate` ile atomic `$inc: soldQty` + `maxTotalQty` kontrol **+ Smart Cap %25 kontrol (§4.3 / barter-rules §5)** — tek bayinin tek siparişi havuz stoğunun %25'ini geçemez |
| `…/application/services/garage-sale-closer.service.ts` | Cron — süre dolunca veya stok bitince `EXHAUSTED/EXPIRED` |
| `…/garage-sale.controller.ts` | REST endpoint |
| `…/domain/garage-sale.entity.ts` | **İndirim fiyat hesabı `Money`/`Decimal` API ile** — `discountedPrice = normalPrice.multiply(Decimal(1).minus(discountRate))`. Float yasak (`financial-services-expert` kuralı). MongoDB'de `Decimal128` olarak saklanır. |
| `…/application/services/smart-cap.service.ts` | **YENİ**: `MAX_POOL_SHARE_PERCENT = 25` sabiti, `validateSmartCap(orderQty, remainingPoolStock)` metodu — ihlal halinde `SMART_CAP_EXCEEDED` fırlatır |

**Sabit**: `MAX_POOL_SHARE_PERCENT = 25` — `shared-core/src/constants/barter.constants.ts` altında merkezi tanım. Standart ekosistem order handler'ı da bu sabiti kullanır (Garaj Günü'ne özgü değil).

**Error codes**: `GARAGE_SALE_STOCK_EXHAUSTED`, `GARAGE_SALE_DEALER_LIMIT_EXCEEDED`, `GARAGE_SALE_NOT_ACTIVE`, `SMART_CAP_EXCEEDED`

**Acceptance**:
- Stok atomic — race condition yok (test: 100 paralel sipariş, sadece `maxTotalQty` kadarı başarılı)
- Süre dolunca otomatik `EXPIRED`, stok bitince `EXHAUSTED`
- `maxQtyPerDealer` Watchover'dan bağımsız uygulanır
- **Smart Cap**: 1000'lik havuzda tek sipariş `> 250` adet ise `SMART_CAP_EXCEEDED`
- **Para hesabı**: Tüm fiyat alanları `Decimal128`; hiçbir yerde `number` (float) tutulmaz; test: `0.1 + 0.2 === 0.3` precision kontrolü
